데이터베이스 DB
Oracle 대표도구 
MySQL무료 dbms 
MS-SQL C#과 호환이 잘된다
질적 데이터 : 취미 특기 등 수치로 나타낼 수 없는 데이터
양적 데이터 : 온도,키 등 수치로 나타낼 수 있는 데이터
데이터는 혼자서 아무것도 아니지만 의미를 부여해주면 정보가 된다
여러개의 테이블을 관리해주는것

필요성
한공간에서 데이터를 효율적으로 관리 저장한다

DBMS : 다수 사용자들이 데이터베이스 내에 접근해 관리 할수록 해주는 SW
대표는 = RDBMS 관계형 데이터베이스
SQL언어를 쓴다 데이터 조작 쉽게 할수있음

코딩과 마찬가지로 처음에 설계가 중요
설계는 E-R다이어그램을 먼저 그린다
ER다이어그램 그릴때는 draw.io 사이트 사용
내보내기로 사진으로 보내기가능
Entity 개체 : 사각형으로 표기한다

키속성
기본키 (PK K 키) 중요!
개체들을 구별할수 있는 유일한 제약조건을 가지는 속성
밑줄을 그어서 표시한다
표에서 PK라고 표시한다

외래키 (FK) 
다른 테이블에 있는 기본키를 참고한다

관계
1;1
1;N 실무에서 많이 사용 됌
M:N
